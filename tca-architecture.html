Financial Domain
Isolated financial boundary · Separate DB · Re-validates every transaction
Bank Service
GnuCOBOL + Go
Player finances. Fixed-point arithmetic in COBOL — no floats near money, ever. Go provides REST API and PostgreSQL layer. Isolated database — no other service has credentials. Re-validates Redis session on every financial operation regardless of JWT. In production, this DB would be an external managed service.
COBOL + Go
financial-net redis-net
User → Service (Trust Boundary)
JWT validated at gateway. OPA policy check per request. Bank operations additionally re-validate Redis session directly — JWT alone is not sufficient for financial operations.
Service → Service (Internal)
Mutual TLS on isolated per-domain bridge networks. Both sides present CA-signed certificates. Connections without a valid client certificate are rejected — not downgraded to unauthenticated.
Communication Domain
Chat Service
Elixir · OTP
E2E encrypted. Public table chat + targeted private messages. OTP supervision trees — a crashed process restarts in isolation, never takes down the game. Designed for exactly this problem (see: WhatsApp).
Elixir
chat-net

Platform Domain
Observability Service
Go · Scratch Container
Receives inter-service event reports from all Jobs (contract obligation: x-tca-observability). Sanitizes (strips UUIDs, JWTs, IPs). Enforces caller allowlist — unknown callers dropped silently. Publishes clean events to Redis tca:events. Fire-and-forget — never on the critical path.
Go
redis-net observability-net
Document Service
TypeScript · PDFKit
Layout engine, not document factory. Accepts structured block data (text, tables, footers) and returns PDF binary. No domain knowledge — any Job can use it for any document type. Stateless: request in, PDF out.
TypeScript
game-net
All 11 other Jobs report outbound calls to Observability Service per x-tca-observability contract obligation. Observability Service itself does not report — it would cause a loop.

Infrastructure
cert-init (Init Container)
Alpine + openssl · Runs once at startup · Generates ephemeral CA + 12 per-service certificates into shared Docker volume · Exits cleanly (restart: "no") · All 12 Jobs depend_on cert-init completing before starting · Volume destroyed on docker compose down · Production: replace with cert-manager or organizational CA
